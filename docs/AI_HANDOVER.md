# 保留（Version3以降で検討）

以下は良いアイデアだが、現状のアーキテクチャ（単一銘柄・都度実行型のStreamlitアプリ）では時期尚早と判断し、いったん保留する。

- シャドーポートフォリオ（見送り銘柄の仮想追跡）：継続的なデータ蓄積とDBが必要
- ポートフォリオ全体のリスク分解（為替・金利感応度など）：複数銘柄管理機能が前提
- ポジションサイズの提案（感情耐久度から算出）：投資助言との境界に注意が必要なため、慎重に設計してから着手する
